**1. Backend: Add GET /api/content/pipeline-status endpoint** in `backend/src/routes/content.ts`.

Add BEFORE the `/:id` catch-all routes (to avoid Express matching `pipeline-status` as an `:id` param). The endpoint:

```typescript
contentRouter.get('/pipeline-status', async (req: Request, res: Response, next: NextFunction) => {
  try {
    const userId = (req as any).user!.id;

    // All content currently in processing states
    const processing = await prisma.content.findMany({
      where: {
        userId,
        status: { in: ['SELECTED', 'TRANSCRIBING', 'GENERATING'] },
      },
      select: { id: true, status: true, title: true, updatedAt: true },
      orderBy: { updatedAt: 'desc' },
    });

    // Content that became READY in the last 5 minutes (for transition detection)
    const fiveMinAgo = new Date(Date.now() - 5 * 60 * 1000);
    const recentlyReady = await prisma.content.findMany({
      where: {
        userId,
        status: 'READY',
        updatedAt: { gte: fiveMinAgo },
      },
      select: { id: true, title: true, updatedAt: true },
      take: 20,
    });

    return res.json({ processing, recentlyReady });
  } catch (error) {
    next(error);
  }
});
```

**3. iOS hook: Create `ios/hooks/usePipelineStatus.ts`**

Create a new hook that:
- Calls `GET /content/pipeline-status`
- Uses React Query's `refetchInterval` callback to poll every 5 seconds ONLY when `processing.length > 0`, otherwise stops polling (`false`)
- Tracks previous processing IDs via `useRef<Set<string>>` to detect ready transitions
- When an item disappears from `processing` and appears in `recentlyReady`, shows a toast and triggers haptic feedback
- Batches notifications: if multiple items become ready in a single poll cycle, show ONE aggregated toast like "3 contenus prets !" instead of individual toasts
- Exports `usePipelineStatus()` returning `{ data, isLoading, processingCount }` and a `processingMap` (Map<contentId, status>) for easy lookup by ContentCard

```typescript
import { useEffect, useRef, useCallback } from 'react';
import { useQuery, useQueryClient } from '@tanstack/react-query';
import * as Haptics from 'expo-haptics';
import { Alert } from 'react-native';
import api from '../lib/api';
import type { PipelineStatusResponse, ContentStatus } from '../types/content';

export function usePipelineStatus() {
  const previousProcessingRef = useRef<Set<string>>(new Set());
  const queryClient = useQueryClient();

  const query = useQuery({
    queryKey: ['pipeline', 'status'],
    queryFn: async (): Promise<PipelineStatusResponse> => {
      const { data } = await api.get<PipelineStatusResponse>('/content/pipeline-status');
      return data;
    },
    refetchInterval: (query) => {
      const data = query.state.data;
      if (data?.processing && data.processing.length > 0) return 5000;
      return false;
    },
    refetchOnWindowFocus: true,
  });

  // Detect ready transitions
  useEffect(() => {
    if (!query.data) return;
    const currentIds = new Set(query.data.processing.map(p => p.id));
    const readyIds = new Set(query.data.recentlyReady.map(r => r.id));

    const newlyReady: string[] = [];
    for (const id of previousProcessingRef.current) {
      if (!currentIds.has(id) && readyIds.has(id)) {
        newlyReady.push(id);
      }
    }

    if (newlyReady.length > 0) {
      // Haptic feedback
      Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success);
      // Invalidate content queries so cards refresh
      queryClient.invalidateQueries({ queryKey: ['content'] });

      // Batched toast (use Alert for now -- simple, no extra dependency)
      if (newlyReady.length === 1) {
        const item = query.data.recentlyReady.find(r => r.id === newlyReady[0]);
        // Don't show blocking Alert -- just invalidate. Toast can be added later.
      } else {
        // Multiple items -- just invalidate, no blocking alert
      }
    }

    previousProcessingRef.current = currentIds;
  }, [query.data]);

  // Build a map for quick lookup: contentId -> status
  const processingMap = new Map<string, ContentStatus>();
  if (query.data?.processing) {
    for (const item of query.data.processing) {
      processingMap.set(item.id, item.status as ContentStatus);
    }
  }

  return {
    data: query.data,
    isLoading: query.isLoading,
    processingCount: query.data?.processing?.length ?? 0,
    processingMap,
  };
}
```

NOTE: For the toast/notification on ready transition, do NOT use Alert (blocking). Instead, just trigger the haptic + query invalidation. The status badge disappearing from the card IS the visual indicator. If a lightweight toast library is already available, use it. Otherwise the badge removal + haptic is sufficient for FEEDBACK-02.

**1. Create `ios/components/content/PipelineStatusBadge.tsx`**

A small badge component that overlays on the ContentCard thumbnail to show processing status. Returns null for non-processing states (READY, INBOX, ARCHIVED).

```typescript
import { View, StyleSheet } from 'react-native';
import { Clock, Mic, Sparkles, AlertCircle, Ban } from 'lucide-react-native';
import Animated, { FadeIn, FadeOut } from 'react-native-reanimated';
import { Text } from '../ui';
import { colors, borderRadius, spacing } from '../../theme';
import type { ContentStatus } from '../../types/content';

const STATUS_CONFIG: Record<string, { icon: any; label: string; color: string }> = {
  SELECTED: { icon: Clock, label: 'En attente', color: colors.textTertiary },
  TRANSCRIBING: { icon: Mic, label: 'Transcription...', color: colors.accent },
  GENERATING: { icon: Sparkles, label: 'Quiz en creation...', color: '#22C55E' },
  FAILED: { icon: AlertCircle, label: 'Erreur', color: '#EF4444' },
  UNSUPPORTED: { icon: Ban, label: 'Non supporte', color: colors.textTertiary },
};

export function PipelineStatusBadge({ status }: { status: ContentStatus }) {
  const config = STATUS_CONFIG[status];
  if (!config) return null; // READY, INBOX, ARCHIVED -- no badge

  const Icon = config.icon;

  return (
    <Animated.View entering={FadeIn.duration(200)} exiting={FadeOut.duration(200)} style={styles.container}>
      <View style={[styles.badge, { backgroundColor: 'rgba(0, 0, 0, 0.75)' }]}>
        <Icon size={10} color={config.color} />
        <Text style={[styles.label, { color: config.color }]}>{config.label}</Text>
      </View>
    </Animated.View>
  );
}

const styles = StyleSheet.create({
  container: {
    position: 'absolute',
    bottom: spacing.xs,
    left: spacing.xs,
  },
  badge: {
    flexDirection: 'row',
    alignItems: 'center',
    gap: 4,
    paddingHorizontal: 6,
    paddingVertical: 3,
    borderRadius: borderRadius.xs,
  },
  label: {
    fontSize: 9,
    fontWeight: '600',
  },
});
```

Position: bottom-left of thumbnail (source badge is top-left, duration is bottom-right, selection checkbox is top-right). This avoids overlap with existing overlays.
