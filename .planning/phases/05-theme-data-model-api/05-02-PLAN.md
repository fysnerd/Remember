**Slug utility (inline, no library):**
```typescript
function generateSlug(name: string): string {
  return name
    .toLowerCase()
    .normalize('NFD')
    .replace(/[\u0300-\u036f]/g, '')
    .replace(/[^a-z0-9\s-]/g, '')
    .replace(/\s+/g, '-')
    .replace(/-+/g, '-')
    .replace(/^-|-$/g, '');
}
```

**Zod validation schemas:**
```typescript
const createThemeSchema = z.object({
  name: z.string().min(1).max(100).trim(),
  color: z.string().regex(/^#[0-9A-Fa-f]{6}$/).optional(),
  emoji: z.string().max(4).optional(),
});

const updateThemeSchema = z.object({
  name: z.string().min(1).max(100).trim().optional(),
  color: z.string().regex(/^#[0-9A-Fa-f]{6}$/).optional(),
  emoji: z.string().max(4).optional(),
});

const addContentSchema = z.object({
  contentIds: z.array(z.string()).min(1).max(100),
});
```

**Endpoint 1: GET / -- List user's themes with content counts (DATA-04)**
- Query: `where: { userId: req.user!.id }`
- Include: `_count: { select: { contentThemes: true } }` and `themeTags: { include: { tag: true } }`
- OrderBy: `{ name: 'asc' }`
- Transform response: map `_count.contentThemes` to `contentCount`, map `themeTags` to flat `tags` array `{ id, name }`
- Return: `{ themes: [...] }`

**Endpoint 2: GET /:id -- Theme detail with paginated content (DATA-05)**
- Params: `id` (theme ID)
- Query params: `page` (default '1'), `limit` (default '20'), `platform` (optional filter: YOUTUBE, SPOTIFY, TIKTOK, INSTAGRAM)
- First: verify theme ownership with `prisma.theme.findFirst({ where: { id, userId } })`
- If not found: return 404
- Build content query through join table: `where: { contentThemes: { some: { themeId } }, userId }` plus optional `platform` filter
- Use `Promise.all([prisma.content.findMany(...), prisma.content.count(...)])` for paginated results
- Include on content: `{ tags: true, _count: { select: { quizzes: true } } }`
- Return: `{ theme: { ...themeData, contentCount, tags }, contents, pagination: { page, limit, total, totalPages } }`

**Endpoint 3: POST / -- Create theme**
- Validate body with `createThemeSchema.parse(req.body)`
- Check theme cap: `prisma.theme.count({ where: { userId } })` >= 25 returns 400
- Generate slug from name
- Check slug uniqueness for user: `prisma.theme.findUnique({ where: { userId_slug: { userId, slug } } })` returns 409 if exists
- Create theme with `prisma.theme.create()`
- Return 201 with theme object
- Catch ZodError for 400 validation errors

**Endpoint 4: PUT /:id -- Update theme**
- Validate body with `updateThemeSchema.parse(req.body)`
- Verify ownership with `findFirst`
- If name changed: regenerate slug and check uniqueness (409 if collision, exclude current theme from check)
- Update with `prisma.theme.update()`
- Return updated theme

**Endpoint 5: DELETE /:id -- Delete theme (preserves content)**
- Verify ownership with `findFirst`
- Delete theme with `prisma.theme.delete()` -- cascade will remove ContentTheme and ThemeTag join records, but NOT the Content or Tag records themselves (those cascade from their own side)
- Return `{ message: 'Theme deleted successfully' }`

**Endpoint 6: POST /:id/content -- Add content items to theme**
- Validate body with `addContentSchema.parse(req.body)`
- Verify theme ownership
- Verify content ownership: `prisma.content.findMany({ where: { id: { in: contentIds }, userId } })`
- Create join records: `prisma.contentTheme.createMany({ data: [...], skipDuplicates: true })`
- Return `{ message, added: result.count }`

**Endpoint 7: DELETE /:id/content/:contentId -- Remove content from theme**
- Verify theme ownership
- Delete the join record: `prisma.contentTheme.deleteMany({ where: { themeId, contentId } })`
- Return `{ message: 'Content removed from theme' }`

**Error handling:** All endpoints use try/catch with `next(error)` following the existing pattern. Zod errors caught specifically for 400 responses.

**Logging:** Use `log.info` for create/delete operations, `log.error` for failures (matching content.ts pattern).
