  <action>
1. **Schema change**: Add two nullable fields to the `Theme` model in `schema.prisma`:
   - `memo String?` -- cached synthesis memo text
   - `memoGeneratedAt DateTime?` -- timestamp for 24h TTL checking
   Place them after the `emoji` field and before `contentThemes`. Then run `npx prisma db push` and `npx prisma generate` locally.

2. **GET /:id/memo endpoint** on `themeRouter` in `themes.ts`:
   - Authenticate via existing middleware (already applied to all themeRouter routes)
   - Look up theme by id + userId (ownership check)
   - **Cache check**: If `theme.memo` exists and `theme.memoGeneratedAt` is less than 24 hours old, return immediately with `{ memo, themeName, contentCount, generatedAt, cached: true }`
   - **Generate**: Query content items in theme via ContentTheme join where `status: 'READY'`, include `transcript`. Extract per-content memos from `transcript.segments.memo` (JSON field). Cap at 15 memos sorted by most recent `capturedAt`.
   - If zero content memos found, return 400 with `{ error: 'Aucun memo disponible pour les contenus de ce theme.', hint: 'Les memos sont generes lors du traitement des quiz.' }`
   - Build synthesis prompt following the topic memo pattern in content.ts lines 1322-1338 but with 400 word max (not 300). System prompt in French.
   - Call `generateText(userPrompt, { system: systemPrompt, temperature: 0.7 })`
   - Save result: `prisma.theme.update({ where: { id: themeId }, data: { memo: synthesized, memoGeneratedAt: new Date() } })`
   - Return `{ memo, themeName, contentCount, generatedAt, cached: false }`
   - Log cache hits/misses with `log.info()`

3. **POST /:id/memo/refresh endpoint** on `themeRouter`:
   - Same logic as GET /:id/memo but always bypasses cache (ignores memoGeneratedAt)
   - Returns `{ memo, themeName, contentCount, generatedAt, regenerated: true }`

4. **Cache invalidation on content change**: In the existing `POST /:id/content` (add content to theme) and `DELETE /:id/content/:contentId` (remove content from theme) endpoints, after the main operation, add a lightweight `prisma.theme.update({ where: { id: themeId }, data: { memo: null, memoGeneratedAt: null } })` to clear the cached memo so the next view triggers fresh generation.

Import `generateText` from `../services/llm.js` at the top of themes.ts.

Define `MEMO_TTL_MS = 24 * 60 * 60 * 1000` as a constant at the top of the file.
