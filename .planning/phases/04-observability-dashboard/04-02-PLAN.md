**Helper functions** (inside the file, before the component):

1. `timeAgo(dateStr: string): string` -- converts ISO date string to human-readable relative time (e.g., "2 min ago", "1 hour ago", "3 days ago"). Use simple math on `Date.now() - new Date(dateStr).getTime()`.

2. `formatDuration(ms: number | null): string` -- converts milliseconds to human-readable duration (e.g., "1.2s", "45.6s", "2m 15s"). Return "-" for null.

3. `statusColor(status: string): string` -- returns badge color: `'success'` for SUCCESS, `'danger'` for FAILED, `'info'` for RUNNING, `'default'` otherwise.

**Section 6: Timeline (DASH-05)**

**SSE Integration (DASH-07):**
- In a `useEffect` with empty deps, create `new EventSource('/admin/api/sse')`
- On `message` event, parse JSON and check `type`:
  - If `job_started`, `job_completed`, or `job_failed` -> call `fetchData()` to refresh the dashboard
- Debounce: if multiple SSE events arrive within 2 seconds, only fetch once. Use a simple `setTimeout`/`clearTimeout` pattern:
  ```tsx
  let refreshTimer: ReturnType<typeof setTimeout> | null = null;
  es.onmessage = (event) => {
    const update = JSON.parse(event.data);
    if (['job_started', 'job_completed', 'job_failed'].includes(update.type)) {
      if (refreshTimer) clearTimeout(refreshTimer);
      refreshTimer = setTimeout(() => fetchData(), 2000);
    }
  };
  ```
- Cleanup: return `() => es.close()` from the useEffect
- Show a small SSE connection indicator (e.g., green dot when connected, red when disconnected) -- use EventSource `onopen` and `onerror` events to track connection state
