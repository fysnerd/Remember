**Step 1: Create `backend/src/utils/slug.ts`**

Extract the `generateSlug` function from `themes.ts` into a shared utility:

```typescript
/**
 * Generate a URL-safe slug from a name.
 * Handles French accents, special characters, and whitespace.
 */
export function generateSlug(name: string): string {
  return name
    .toLowerCase()
    .normalize('NFD')
    .replace(/[\u0300-\u036f]/g, '')
    .replace(/[^a-z0-9\s-]/g, '')
    .replace(/\s+/g, '-')
    .replace(/-+/g, '-')
    .replace(/^-|-$/g, '');
}
```

**Step 3: Create `backend/src/services/themeClassification.ts`**

Create the complete theme classification service following the existing `tagging.ts` worker pattern. The file must export these functions:

1. **`runThemeClassificationWorker()`** -- Main worker entry point (called by scheduler):
   - Stage A: Find users with 0 themes AND 10+ tagged content items. Process each with `pLimit(3)` calling `generateThemesForUser()`.
   - Stage B: Find content items that have tags but no `contentThemes` records AND whose user has existing themes. Take 20 at a time, process each with `pLimit(5)` calling `classifyContentForUser()`.
   - Wrap entire function in try/catch, log start/completion/errors.

2. **`generateThemesForUser(userId: string)`** -- Theme generation:
   - Check if user already has themes (`prisma.theme.count`). If yes, skip (return early). This ensures idempotency (CLASS-04, success criterion #4).
   - Query user's tags with usage counts: `SELECT t.id, t.name, COUNT(ct."A") as count FROM "Tag" t JOIN "_ContentTags" ct ON ct."B" = t.id JOIN "Content" c ON c.id = ct."A" WHERE c."userId" = $userId GROUP BY t.id, t.name ORDER BY count DESC`. Use `prisma.$queryRaw` for this aggregation.
   - Filter to tags with count >= 2 (drop single-use tags per research recommendation).
   - If fewer than 5 tags remain after filtering, skip (not enough signal).
   - Call `generateThemesFromTags()` with the tag list.
   - For each returned theme from LLM: generate slug, check uniqueness against existing themes (should be none since we checked), enforce MAX_THEMES=25 cap, create Theme record + ThemeTag records in a `prisma.$transaction`.
   - After creating themes, call `classifyAllContentForUser()` to assign existing content.

3. **`classifyContentForUser(contentId: string, userId: string)`** -- Single content classification:
   - Load content's tags (`prisma.content.findUnique` with tags include).
   - Load user's themes with their associated tags (`prisma.theme.findMany` where userId, include themeTags -> tag).
   - **Deterministic matching first:** For each theme, check if any of the content's tag IDs appear in the theme's ThemeTag tagIds. If overlap found, that theme is a match.
   - If deterministic matching finds 1+ themes, use those matches. Create `ContentTheme` records with `createMany({ skipDuplicates: true })` and `assignedBy: 'system'`.
   - **LLM fallback** if no deterministic match: Call `classifyContentViaLLM()` passing content title, content tag names, and available themes with their tag names. Parse returned theme names, match to theme IDs, create ContentTheme records.

4. **`runBackfillThemes()`** -- One-time backfill:
   - Find all users with `contents.some.tags.some` and count tagged content >= 10.
   - For each eligible user: if 0 themes, call `generateThemesForUser()`. Then classify all unthemed tagged content via `classifyAllContentForUser()`.
   - Use `pLimit(2)` for user-level concurrency during backfill (lower than normal to be gentle on LLM API).

5. **Internal helpers (not exported):**

   - **`generateThemesFromTags(userTags, existingThemes)`**: LLM prompt that clusters tags into themes. Prompt in French. System prompt instructs: 5-15 themes, merge synonyms/bilingual duplicates, French names, 2-4 words max, include emoji and hex color from fixed palette (`#EF4444, #F97316, #EAB308, #22C55E, #14B8A6, #3B82F6, #6366F1, #8B5CF6, #EC4899, #F43F5E, #06B6D4, #84CC16`), include associated tag names. Temperature 0.3, maxTokens 2000, jsonMode true. Parse JSON response with try/catch, validate structure.

   - **`classifyContentViaLLM(contentTitle, contentTags, existingThemes)`**: Lighter LLM call. Ask which 1-3 existing themes match this content. Temperature 0.2, maxTokens 200, jsonMode true. Return theme names array.

   - **`classifyAllContentForUser(userId)`**: Find all content for user with tags but no contentThemes. Process each with `pLimit(5)` calling `classifyContentForUser()`.
