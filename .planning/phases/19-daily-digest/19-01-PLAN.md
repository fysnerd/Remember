  <action>
Add a new `POST /reviews/digest` endpoint to `reviewRouter` in `backend/src/routes/review.ts`. Place it after the existing `/stats` endpoint (after line ~382) and before the submit review schema.

The endpoint implements this algorithm:

1. **SRS due cards (priority):** Query `prisma.card.findMany` where `userId`, `repetitions > 0`, `nextReviewAt <= new Date()`, ordered by `nextReviewAt ASC` (most overdue first), limited to 15 (`MAX_DIGEST`). Include `quiz` with nested `content` (select: `id`, `title`, `url`, `platform`). Also include `quiz.isSynthesis` and `quiz.theme` (select: `id`, `name`).

2. **New card budget:** Look up `userSettings.newCardsPerDay` (default 20). Count today's first reviews (`review.count` where `createdAt >= today`, `card.repetitions === 1`). Calculate `remainingNewToday = max(0, newCardsPerDay - firstReviewsToday)`. Calculate `newSlots = min(max(0, MAX_DIGEST - dueCards.length), remainingNewToday)`.

3. **New cards (fill):** If `newSlots > 0`, query `prisma.card.findMany` where `userId`, `repetitions === 0`, `nextReviewAt <= new Date()`, ordered by `createdAt ASC` (oldest first), limited to `newSlots`. Same include structure as due cards.

4. **Combine and shuffle:** Merge both arrays and shuffle with `sort(() => Math.random() - 0.5)`.

5. **Empty check:** If combined array is empty, return `{ cards: [], session: null, reason: 'no_cards_due', count: 0, stats: { dueCount: 0, newCount: 0 } }`.

6. **Auto-create session:** `prisma.quizSession.create({ data: { userId, questionLimit: allCards.length, mode: 'due' } })`.

7. **Return:** `{ cards: allCards, count: allCards.length, session: { id: session.id }, stats: { dueCount: dueCards.length, newCount: newCards.length } }`.

Use constants `MIN_DIGEST = 10` and `MAX_DIGEST = 15` defined at the top of the handler (inline, not module-level, per 17-01 decision pattern). The MIN_DIGEST is informational only (no hard minimum enforced -- return whatever is available per research recommendation).

Mirror the existing include structure from the `/reviews/due` endpoint (lines ~183-278) for the card queries, adapting as needed for the digest response shape. Wrap in try/catch with `next(error)` like all other endpoints.

**Create `ios/hooks/useDigest.ts`:**

Define TypeScript interfaces matching the backend response:

```typescript
interface DigestCard {
  id: string; // card.id
  quiz: {
    id: string;
    question: string;
    options: string[] | Record<string, string>;
    correctAnswer: string;
    explanation: string;
    content: { id: string; title: string; url: string; platform: string } | null;
    isSynthesis?: boolean;
    theme?: { id: string; name: string };
  };
}

interface DigestResponse {
  cards: DigestCard[];
  count: number;
  session: { id: string } | null;
  reason?: string;
  stats: { dueCount: number; newCount: number };
}
```

Export a `useDigestCards` hook using `useMutation` from `@tanstack/react-query`:

```typescript
export function useDigestCards() {
  return useMutation({
    mutationFn: async () => {
      const { data } = await api.post<DigestResponse>('/reviews/digest');
      return data;
    },
  });
}
```

Use `useMutation` (NOT `useQuery`) because the endpoint creates a QuizSession as a side effect. This prevents accidental re-fetches (staleTime, focus refetch, etc.) from creating duplicate sessions.

Import `api` from `../lib/api` and `useMutation` from `@tanstack/react-query`.
