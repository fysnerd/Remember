**Create `ios/app/digest.tsx`:**

Full digest session screen managing its own state machine internally. Reference the existing `ios/app/quiz/[id].tsx` for the question/feedback pattern, but adapted for the digest flow.

**State machine:** `'loading' | 'question' | 'feedback' | 'closure'`

**Local state:**
```typescript
const [phase, setPhase] = useState<'loading' | 'question' | 'feedback' | 'closure'>('loading');
const [currentIndex, setCurrentIndex] = useState(0);
const [selectedAnswer, setSelectedAnswer] = useState<string | null>(null);
const [isCorrect, setIsCorrect] = useState(false);
const [score, setScore] = useState(0);
const [currentStreak, setCurrentStreak] = useState(0);
const [bestStreak, setBestStreak] = useState(0);
const [sessionStartTime] = useState(Date.now());
```

**On mount:** Trigger `digestMutation.mutate()` via `useEffect` (runs once). When mutation succeeds (`onSuccess` callback or `useEffect` watching `digestMutation.data`):
- If `data.cards.length === 0` or `data.session === null`, show empty state with message "Rien a reviser aujourd'hui !" and a "Retour" button that calls `router.back()`.
- Otherwise, set `phase` to `'question'`.

**Question phase:**
- Render `ProgressBar` at the top with `current={currentIndex}` and `total={cards.length}`.
- Render the existing `QuestionCard` component with the current card's quiz data.
- QuestionCard handles option selection. When user taps "Valider" (or equivalent), call `submitAnswer.mutateAsync({ cardId, sessionId, rating, responseTime })`.
- Determine `isCorrect` from the answer (compare selected answer to `quiz.correctAnswer`).
- Update streak tracking:
  ```typescript
  if (correct) {
    const newStreak = currentStreak + 1;
    setCurrentStreak(newStreak);
    if (newStreak > bestStreak) setBestStreak(newStreak);
    setScore(prev => prev + 1);
  } else {
    setCurrentStreak(0);
  }
  ```
- Trigger haptic feedback: `haptics.success()` on correct, `haptics.error()` on incorrect.
- Transition to `'feedback'` phase.

**Feedback phase:**
- Render `AnswerFeedback` component with the correct answer and explanation.
- "Suivant" button: If `currentIndex < cards.length - 1`, increment `currentIndex`, reset `selectedAnswer`, set `phase` to `'question'`.
- If `currentIndex === cards.length - 1` (last question), button text changes to "Voir le resultat". On press, call `completeSession.mutateAsync(sessionId)`, then set `phase` to `'closure'`.

**Closure phase:**
- Render `DigestClosure` with `score`, `total={cards.length}`, `bestStreak`, `durationMs={Date.now() - sessionStartTime}`, `onClose={() => router.back()}`.

**Navigation header:** Add a top-left "Quitter" button (X icon or text) that navigates back. Use `SafeAreaView` with edges `['top']` and a header row with the quit button and the progress indicator.

**Imports:** `useDigestCards` from hooks, `useSubmitAnswer` and `useCompleteSession` from `../hooks/useQuiz`, `QuestionCard` from `../components/quiz/QuestionCard`, `AnswerFeedback` from `../components/quiz/AnswerFeedback`, `ProgressBar` from `../components/digest/ProgressBar`, `DigestClosure` from `../components/digest/DigestClosure`, `haptics` from `../lib/haptics`, `router` from `expo-router`, design tokens from `../theme`.

**Important considerations:**
- Do NOT show a live timer during the session (causes anxiety per research). Only compute duration at closure time.
- Disable "Valider" button while `!sessionId` (prevents race condition between session creation and first answer -- per pitfall 5 from research).
- Use the same option normalization logic as `quiz/[id].tsx` for handling both array and object option formats from the backend.
- Background color: `colors.background` (#0A0F1A).
