**1. Create `ios/app/theme/manage/[id].tsx`** -- Follow `topic/manage/[name].tsx` pattern closely:

**Rename section:**
- TextInput pre-filled with current theme name. Save button enabled when name changed.
- On save: call `useUpdateTheme().mutateAsync({ id, name: newName.trim() })`.
- On success: `router.replace({ pathname: '/theme/[id]', params: { id } })`.
- On error: `Alert.alert('Erreur', 'Impossible de renommer le theme.')`.

**Emoji/Color editing section (simple):**
- Show current emoji and color. Provide two sub-sections:
- **Emoji palette:** A horizontal `ScrollView` or wrapped `View` of ~20 preset emojis: `['📚', '🎵', '💻', '🧠', '🎮', '🏋️', '💰', '🎨', '🌍', '🔬', '📊', '🏥', '🎬', '🍳', '✈️', '📱', '🔧', '💡', '🎯', '❤️']`. Tapping one updates local state. Save button commits with `useUpdateTheme`.
- **Color palette:** A row of 12 tappable circles using the same palette from the AI worker: `['#EF4444', '#F97316', '#EAB308', '#22C55E', '#14B8A6', '#3B82F6', '#6366F1', '#8B5CF6', '#EC4899', '#F43F5E', '#06B6D4', '#84CC16']`. Tapping one updates local state. Same save button.
- Combine emoji + color + name updates into a single save action.

**Content list section:**
- Use `useThemeDetail(id!)` to get content list.
- Render each content item with source emoji, title, and a red "X" remove button on the right.
- On remove press: call `useRemoveContentFromTheme().mutateAsync({ themeId: id, contentId })`.
- Each item is also tappable to navigate to `/content/[id]`.

**Danger zone section:**
- Same pattern as `topic/manage/[name].tsx`.
- `Alert.alert()` confirmation dialog with destructive "Supprimer" option.
- On confirm: `useDeleteTheme().mutateAsync(id!)` then `router.replace('/(tabs)')`.

Use `Stack.Screen` with `title: 'Gerer le theme'`, `headerBackTitle: 'Retour'`.
