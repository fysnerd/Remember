  <name>Task 1: Migrate transcription services to Pino (4 files, 141 occurrences)</name>
  <files>backend/src/services/transcription.ts, backend/src/services/podcastTranscription.ts, backend/src/services/tiktokTranscription.ts, backend/src/services/instagramTranscription.ts</files>
  <action>
    These 4 files are the highest-volume logging in the codebase (141 of 218 service occurrences). They handle content transcription via yt-dlp and Whisper.

    For EACH file:

    1. Add import: `import { logger } from '../config/logger.js';`
    2. Create child logger at module level:
       - transcription.ts: `const log = logger.child({ service: 'youtube-transcription' });`
       - podcastTranscription.ts: `const log = logger.child({ service: 'podcast-transcription' });`
       - tiktokTranscription.ts: `const log = logger.child({ service: 'tiktok-transcription' });`
       - instagramTranscription.ts: `const log = logger.child({ service: 'instagram-transcription' });`

    3. Replace ALL console.log/error following these mapping rules:

       **Business events (info level):**
       - Starting worker run -> `log.info('Starting transcription worker');`
       - Found N pending items -> `log.info({ count: N }, 'Found pending transcriptions');`
       - Successfully transcribed -> `log.info({ contentId, platform, durationMs }, 'Transcription completed');`
       - Worker completed -> `log.info({ processed: N, failed: M }, 'Transcription worker completed');`

       **Operational detail (debug level):**
       - Downloading audio -> `log.debug({ contentId, url }, 'Downloading audio');`
       - Calling Whisper API -> `log.debug({ contentId, audioSizeKb }, 'Sending to Whisper');`
       - Subtitle extraction -> `log.debug({ contentId, method: 'yt-dlp' }, 'Extracting subtitles');`

       **Failures (error level):**
       - Transcription failed -> `log.error({ err: error, contentId }, 'Transcription failed');`
       - API error -> `log.error({ err: error, contentId, api: 'groq' }, 'Whisper API error');`
       - Download failed -> `log.error({ err: error, contentId, url }, 'Audio download failed');`

       **Warnings (warn level):**
       - Max retries exceeded -> `log.warn({ contentId, retries: N }, 'Max retries exceeded, marking as failed');`
       - Content skipped -> `log.warn({ contentId, reason }, 'Skipping content');`

    4. Key context fields to always include when available:
       - `contentId` -- identifies which content item
       - `userId` -- identifies which user's content
       - `platform` -- youtube/spotify/tiktok/instagram
       - `durationMs` -- processing time
       - `err` -- error object (use Pino's `err` key for automatic serialization)

    5. Ensure ZERO console.log/error remain in each file.
  </action>
