**Gap 2 (major): index.tsx - Discovery banner invisible + pull-to-refresh freeze**

Issue 1: Banner conditional on `pendingCount > 0` (line 72), but pendingThemes may be empty array even when themes exist.

Debug fix: Log pendingThemes in development to verify data:

```tsx
const pendingCount = pendingThemes?.length ?? 0;

```

Issue 2: onRefresh (lines 38-45) has no error handling - Promise.all can throw and freeze UI.

Fix: Wrap in try/catch/finally:
```tsx
const onRefresh = useCallback(async () => {
  setRefreshing(true);
  try {
    await Promise.all([
      queryClient.invalidateQueries({ queryKey: ['themes'] }),
      queryClient.invalidateQueries({ queryKey: ['reviews', 'stats'] }),
    ]);
  } catch (error) {
    console.error('[Home] Refresh error:', error);
  } finally {
    setRefreshing(false);
  }
}, [queryClient]);
```

**Gap 3 (major): SelectionBar.tsx - Hidden behind tab bar**

**Gap 4 (blocker): CategoryChips.tsx - Active chip background stretches full height**

Issue: Chip Pressable (line 37-41) has no height constraint - the accent background fills available vertical space because ScrollView parent has no height and chips lack `alignSelf`.

Fix: Add `alignSelf: 'flex-start'` to chip style (line 74):
```tsx
chip: {
  paddingVertical: spacing.xs,
  paddingHorizontal: spacing.md,
  borderRadius: borderRadius.full,
  alignSelf: 'flex-start', // NEW: Prevent vertical stretching
},
```

This constrains the Pressable to wrap its content height instead of filling parent.
