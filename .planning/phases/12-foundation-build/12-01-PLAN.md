**Step 1: Update `ios/components/ui/Text.tsx`** -- switch from fontWeight to fontFamily for Geist:

Replace `variantStyles` to use `fontFamily` from the `fonts` token (NOT `fontWeight`). This is critical because React Native ignores `fontWeight` on custom fonts:
```typescript
const variantStyles: Record<TextVariant, TextStyle> = {
  h1: { fontSize: 28, fontFamily: fonts.bold, lineHeight: 36 },
  h2: { fontSize: 24, fontFamily: fonts.bold, lineHeight: 32 },
  h3: { fontSize: 20, fontFamily: fonts.semibold, lineHeight: 28 },
  body: { fontSize: 16, fontFamily: fonts.regular, lineHeight: 24 },
  caption: { fontSize: 14, fontFamily: fonts.regular, lineHeight: 20 },
  label: { fontSize: 12, fontFamily: fonts.medium, lineHeight: 16 },
};
```

Add `'semibold'` to the `TextWeight` type:
```typescript
type TextWeight = 'regular' | 'medium' | 'semibold' | 'bold';
```

Replace `weightMap` to return fontFamily strings instead of fontWeight strings:
```typescript
const weightMap: Record<TextWeight, string> = {
  regular: fonts.regular,
  medium: fonts.medium,
  semibold: fonts.semibold,
  bold: fonts.bold,
};
```

In the component JSX, change the `weight` style application from `{ fontWeight: weightMap[weight] }` to `{ fontFamily: weightMap[weight] }`:

Remove `fontWeight` from `variantStyles` entirely -- do NOT have both `fontWeight` and `fontFamily` on the same style, as it causes unpredictable behavior with custom fonts.
