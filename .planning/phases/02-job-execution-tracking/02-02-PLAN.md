**Change 2: Modify the `runJob()` function to wrap job execution with tracking**

Replace the existing `runJob()` function body. The tracking wrapper goes INSIDE the runJob function, wrapping `await job()`. The existing overlap prevention (`runningJobs` Set) remains untouched -- tracking happens AFTER the overlap check passes.

Current `runJob()`:
```typescript
async function runJob(jobName: string, job: () => Promise<void>): Promise<void> {
  if (runningJobs.has(jobName)) {
    log.warn({ job: jobName }, 'Skipping job - previous run still in progress');
    return;
  }

  runningJobs.add(jobName);
  const startTime = Date.now();
  try {
    await job();
    const duration = Date.now() - startTime;
    log.info({ job: jobName, durationMs: duration }, 'Job completed');
  } catch (error) {
    log.error({ job: jobName, err: error }, 'Job execution failed');
  } finally {
    runningJobs.delete(jobName);
  }
}
```

Replace with:
```typescript
async function runJob(jobName: string, job: () => Promise<void>): Promise<void> {
  if (runningJobs.has(jobName)) {
    log.warn({ job: jobName }, 'Skipping job - previous run still in progress');
    return;
  }

  runningJobs.add(jobName);
  try {
    await trackJobExecution(jobName, job);
  } catch (error) {
    // Error already logged by trackJobExecution and persisted to DB.
    // Log here too for PM2 log visibility.
    log.error({ job: jobName, err: error }, 'Job execution failed');
  } finally {
    runningJobs.delete(jobName);
  }
}
```

Key changes explained:
- Removed inline `startTime` / `duration` tracking -- `trackJobExecution` handles timing and persistence now
- Removed the success log `log.info(...)` -- `trackJobExecution` logs completion internally (avoids duplicate logs)
- The catch block still logs at `runJob` level for PM2 log visibility, but error is also captured in DB by `trackJobExecution`
- `runningJobs.add/delete` and overlap check remain exactly as before -- tracking is orthogonal to overlap prevention
