# Requirements: Ankora v4.0

**Defined:** 2026-02-16
**Core Value:** L'utilisateur apprend durablement a partir de ce qu'il consomme deja -- sans effort supplementaire de curation.

## v4.0 Requirements

Requirements for the UX Triage & Daily Digest milestone. All 22 requirements shipped.

### Triage

- [x] **TRIAGE-01**: User can swipe right on inbox card to keep content (triggers learn/classify pipeline)
- [x] **TRIAGE-02**: User can swipe left on inbox card to dismiss content (archives it)
- [x] **TRIAGE-03**: User can toggle between swipe mode and bulk select mode via top-right button
- [x] **TRIAGE-04**: User can select/deselect multiple items in bulk mode and batch learn/archive
- [x] **TRIAGE-05**: User can filter inbox by source platform (YouTube, Spotify, TikTok, Instagram) in both modes
- [x] **TRIAGE-06**: User can pull-to-refresh to trigger platform sync in swipe mode
- [x] **TRIAGE-07**: Inbox cards display sorted most recent first (capturedAt desc)
- [x] **TRIAGE-08**: Swipe actions show animated visual feedback (card flies off-screen with spring physics)

### SRS

- [x] **SRS-01**: First review is scheduled 24h after content triage (J+1), not immediately
- [x] **SRS-02**: Review intervals follow J+1, J+3, J+7, J+31 fixed progression
- [x] **SRS-03**: Failed review (rating < 3) resets card to J+1 interval
- [x] **SRS-04**: Card easeFactor adjusts based on user performance (SM-2 compatible)

### Quiz

- [x] **QUIZ-01**: Quiz questions include creator/channel name in question text
- [x] **QUIZ-02**: Quiz questions reference source platform context (e.g., "dans cette video YouTube de [creator]")
- [x] **QUIZ-03**: Quiz generation prompt uses self-referential framing with temporal context

### Digest

- [x] **DIGEST-01**: User can launch a daily digest session from home screen
- [x] **DIGEST-02**: Daily digest pre-selects 10-15 questions mixing due SRS cards and new cards
- [x] **DIGEST-03**: User sees question-by-question progress during digest (e.g., "Question 5/12")
- [x] **DIGEST-04**: User sees cognitive closure screen at end of session with stats summary (score, streak, time)
- [x] **DIGEST-05**: Daily digest prioritizes SRS due cards over new content cards

### Feedback

- [x] **FEEDBACK-01**: User sees real-time processing status per content (transcribing, generating, ready)
- [x] **FEEDBACK-02**: User receives visual indicator when new content becomes quiz-ready after triage
