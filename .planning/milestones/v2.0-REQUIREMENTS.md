## v2.0 Requirements (All Shipped)

### Data Model & API (Phase 5)

- [x] **DATA-01**: Theme model exists in database with name, slug, color, emoji, per-user ownership
- [x] **DATA-02**: Content can belong to multiple themes via many-to-many relation (explicit join table)
- [x] **DATA-03**: Themes are linked to tag patterns for deterministic classification
- [x] **DATA-04**: API endpoint returns user's themes with content counts
- [x] **DATA-05**: API endpoint returns theme detail with its content list (paginated, filterable by source)
- [x] **DATA-06**: Content API responses include associated themes

### Classification (Phase 6)

- [x] **CLASS-01**: Worker auto-generates themes from user's existing tags via Mistral AI (batch clustering)
- [x] **CLASS-02**: Worker auto-classifies new content into existing themes when auto-tagging completes
- [x] **CLASS-03**: Existing content is backfilled into themes via one-time migration job
- [x] **CLASS-04**: Theme creation passes existing themes to LLM to prevent duplicates/near-duplicates
- [x] **CLASS-05**: Themes are capped at 15-25 per user to prevent proliferation

### Navigation - iOS (Phase 7)

- [x] **NAV-01**: Home screen displays theme sections with recent content per theme
- [x] **NAV-02**: User can tap a theme to see all its content (theme detail screen)
- [x] **NAV-03**: Theme cards show name, emoji, color, and content count
- [x] **NAV-04**: Theme list replaces current tag-based "Topics" as primary navigation
- [x] **NAV-05**: User can pull-to-refresh to update theme data

### Management (Phase 7)

- [x] **MGMT-01**: User can rename a theme
- [x] **MGMT-02**: User can delete a theme (content preserved, only grouping removed)
- [x] **MGMT-03**: User can move content between themes (add/remove theme assignment)
- [x] **MGMT-04**: User can create a new theme manually (optional, AI-first but user can override)

### Quiz (Phases 8, 10)

- [x] **QUIZ-01**: User can start a quiz scoped to a specific theme (mix of existing per-content questions)
- [x] **QUIZ-02**: Theme quiz mixes questions from all content items within the theme
- [x] **QUIZ-03**: AI generates new synthesis questions that connect concepts across multiple content items in a theme
- [x] **QUIZ-04**: Synthesis questions require knowledge from at least 2 different content sources
- [x] **QUIZ-05**: Theme quiz requires minimum 3 content items with quizzes before enabling

### Memo (Phase 9)

- [x] **MEMO-01**: API generates a theme-level synthesis memo aggregating knowledge from all content in a theme
- [x] **MEMO-02**: User can view theme memo from theme detail screen
- [x] **MEMO-03**: Theme memo is cached and refreshed on-demand (24h TTL)

### Onboarding & Discovery (Phase 11)

- [x] **DISC-01**: First-time theme discovery flow shows AI-generated themes for user review
- [x] **DISC-02**: User can rename, merge, or dismiss themes during onboarding
- [x] **DISC-03**: Theme learning progress visible on theme cards (mastery %, cards due)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Hierarchical/nested themes | Prisma lacks recursive queries, UX complexity, 5-15 themes not enough for hierarchy |
| Manual-only theme creation | Undermines auto-classification value prop |
| Theme sharing between users | Social features, privacy, multi-tenant complexity |
| Real-time theme updates (WebSocket) | Themes change slowly, React Query polling sufficient |
| Theme-based content recommendations | Requires recommendation engine, out of scope |
| Vector database for classification | Tag space too small (50-200/user), LLM classification simpler |
| Theme-specific SM-2 schedules | SM-2 operates per-card, overriding intervals fights the algorithm |
